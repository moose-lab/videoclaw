  三、核心模块职责定义

  1. Director（导演 Agent）

  整个系统的"大脑"。接收用户意图，拆解为可执行的生产计划。
  ┌──────────┬──────────────────────────────────────────────────┐
  │   职责   │                       说明                       │
  ├──────────┼──────────────────────────────────────────────────┤
  │ 意图理解 │ 解析用户输入（一句话/脚本/参考视频）为结构化需求 │
  ├──────────┼──────────────────────────────────────────────────┤
  │ 生产规划 │ 调用 LLM 生成完整的视频生产方案                  │
  ├──────────┼──────────────────────────────────────────────────┤
  │ 模型选择 │ 根据场景、质量、成本自动选择最优模型组合         │
  ├──────────┼──────────────────────────────────────────────────┤
  │ 质量审查 │ 对生成结果进行自动评审，决定是否需要重试         │
  ├──────────┼──────────────────────────────────────────────────┤
  │ 异常处理 │ 模型失败时的降级策略和路由切换                   │
  └──────────┴──────────────────────────────────────────────────┘
  用户: "做一个30秒的产品介绍视频，科技风格"
           │
           ▼
  Director 输出:
    ├── 脚本: 5段文案 (LLM生成)
    ├── 分镜: 6个镜头 (每镜头5秒)
    ├── 风格: cyberpunk, 深蓝主色调
    ├── 配音: 中文男声, 沉稳风格
    ├── 配乐: 电子音乐, 120BPM
    └── 模型分配:
         ├── 镜头1-3: Sora API (复杂运镜)
         ├── 镜头4-5: CogVideo (产品特写)
         └── 镜头6: AnimateDiff (Logo动画)

  2. Planner（任务规划器）

  将 Director 的方案转化为 DAG（有向无环图） 任务流。

      │          │          │
      └──────────┼──────────┘

           ┌───────────┐

  关键设计：每个节点都是幂等的、可重试的、可检查点恢复的。

  3. DAG Executor（流水线引擎）
  ┌──────────┬─────────────────────────────────────────┐
  │   特性   │                  说明                   │
  ├──────────┼─────────────────────────────────────────┤
  │ 并行调度 │ 无依赖的节点自动并行执行                │
  ├──────────┼─────────────────────────────────────────┤
  │ 断点续跑 │ 任何节点失败后可从该点恢复              │
  ├──────────┼─────────────────────────────────────────┤
  │ 资源感知 │ 本地 GPU 忙时自动路由到云端 API         │
  ├──────────┼─────────────────────────────────────────┤
  │ 回调通知 │ 每个节点完成后触发事件（WebSocket推送） │
  ├──────────┼─────────────────────────────────────────┤
  │ 超时控制 │ 每个节点有独立超时和重试策略            │
  └──────────┴─────────────────────────────────────────┘

  五、模型适配接口规范

  这是整个系统的可扩展性基石。采用
  Protocol（结构子类型）而非抽象基类，降低接入门槛。

  # src/videoclaw/models/protocol.py

  from typing import Protocol, AsyncIterator, runtime_checkable
  from dataclasses import dataclass
  from enum import Enum

  class ModelCapability(Enum):
      TEXT_TO_VIDEO = "t2v"
      IMAGE_TO_VIDEO = "i2v"
      VIDEO_TO_VIDEO = "v2v"
      TEXT_TO_IMAGE = "t2i"
      INPAINTING = "inpaint"
      UPSCALE = "upscale"

  class ExecutionMode(Enum):
      LOCAL = "local"       # 本地GPU推理
      CLOUD = "cloud"       # 云端API调用
      HYBRID = "hybrid"     # 自动选择

  @dataclass
  class GenerationRequest:
      prompt: str
      negative_prompt: str = ""
      width: int = 1280
      height: int = 720
      duration_seconds: float = 5.0
      fps: int = 24
      seed: int | None = None
      reference_image: bytes | None = None    # i2v场景
      reference_video: bytes | None = None    # v2v场景
      style_preset: str | None = None
      extra: dict = field(default_factory=dict)  # 模型特定参数

  @dataclass
  class GenerationResult:
      video_data: bytes                    # 视频二进制
      format: str = "mp4"
      duration_seconds: float = 0.0
      metadata: dict = field(default_factory=dict)
      cost_usd: float = 0.0               # 本次调用成本
      model_id: str = ""

  @dataclass
  class ProgressEvent:
      progress: float          # 0.0 ~ 1.0
      stage: str               # "queued" | "generating" | "encoding"
      preview_frame: bytes | None = None   # 中间预览帧

  @runtime_checkable
  class VideoModelAdapter(Protocol):
      """任何视频模型只需实现这个协议即可接入 VideoClaw"""

      @property
      def model_id(self) -> str:
          """唯一标识符，如 'sora-v1', 'cogvideo-x-5b'"""
          ...

      @property
      def capabilities(self) -> list[ModelCapability]:
          """声明支持的能力"""
          ...

      @property
      def execution_mode(self) -> ExecutionMode:
          """声明执行模式"""
          ...

      async def generate(self, request: GenerationRequest) -> GenerationResult:
          """核心生成方法"""
          ...

      async def generate_stream(
          self, request: GenerationRequest
      ) -> AsyncIterator[ProgressEvent | GenerationResult]:
          """流式生成（支持进度回调）"""
          ...

      async def estimate_cost(self, request: GenerationRequest) -> float:
          """成本预估（USD），用于智能路由决策"""
          ...

      async def health_check(self) -> bool:
          """健康检查，用于故障转移"""
          ...

  六、技术栈建议
  层级: 语言
  技术: Python 3.12+
  理由: AI生态核心，异步支持成熟
  ────────────────────────────────────────

  理由: 极快，已成为Python社区新标准

  理由: 异步原生、自动文档、WebSocket支持
  ────────────────────────────────────────

  ────────────────────────────────────────

  理由: 工业标准，无可替代
  ────────────────────────────────────────
  层级: 前端
  技术: Next.js 15 + Tailwind + shadcn/ui
  理由: 现代前端标配
  ────────────────────────────────────────
  层级: 实时通信
  技术: WebSocket (FastAPI native)
  理由: 生成进度实时推送
  ────────────────────────────────────────
  层级: 容器化
  技术: Docker Compose
  理由: 一键启动所有依赖
  ────────────────────────────────────────
  层级: 本地推理
  技术: ComfyUI API / diffusers
  理由: 复用成熟的本地推理生态
  ────────────────────────────────────────
  层级: LLM接入
  技术: LiteLLM
  理由: 一个接口兼容100+模型
  ────────────────────────────────────────
  层级: 配置
  技术: Pydantic Settings + YAML
  理由: 类型安全 + 人类可读
  为什么不用 Celery 做核心？

  Celery 适合: 独立、同构的任务队列
  VideoClaw 需要: DAG编排、条件分支、断点续跑、资源感知路由

  自研轻量DAG引擎 + asyncio 提供:
    ✓ 零外部依赖启动 (pip install videoclaw && videoclaw run)
    ✓ DAG原生支持（并行/串行/条件）
    ✓ 检查点恢复
    ✓ 进程内调度（开发）+ 分布式调度（生产）可切换

  Director: "调整 prompt，增加 golden hour lighting"
       │
       ▼
  Cameraman → Sora API → 新结果
       │
       ▼
  Reviewer: "质量合格，打分 8/10，通过"

  最大3轮重试，每轮 prompt 由 LLM 自动优化。不是盲目重试，是带推理的迭代。

  成本计算核心逻辑：

  @dataclass
  class CostRecord:
      task_id: str
      model_id: str
      execution_mode: str          # "local" | "cloud"
      api_cost_usd: float          # API调用费用
      compute_cost_usd: float      # 本地计算费用（GPU电费估算）
      duration_seconds: float      # 执行耗时
      input_tokens: int = 0        # LLM输入token
      output_tokens: int = 0       # LLM输出token
      video_seconds: float = 0.0   # 生成视频时长
      retries: int = 0             # 重试次数（计入总成本）
      timestamp: datetime

  class CostTracker:
      """每个Project内嵌的成本追踪器"""

      def track(self, record: CostRecord) -> None: ...
      def get_project_total(self) -> ProjectCostSummary: ...
      def get_optimization_hints(self) -> list[CostHint]: ...
      def estimate_remaining(self, pending_tasks) -> float: ...
      def check_budget(self, budget_usd: float) -> bool: ...

  差异化功能：预算守卫

  # project config
  budget:
    max_usd: 5.00
    strategy: pause_and_ask    # pause_and_ask | switch_to_local | hard_stop
    alert_at: 80%              # 80%时预警

  当成本接近预算时：
  - pause_and_ask: 暂停并询问用户
  - switch_to_local: 自动切换到本地模型继续
  - hard_stop: 硬停止

  一句话定位：

  与 ComfyUI 的关系：互补而非竞争

  VideoClaw = 完整视频的端到端生产编排（任务级节点）

  VideoClaw 可以内嵌 ComfyUI 作为一个 Model Adapter。
  用户在 VideoClaw 的一个节点里调用整个 ComfyUI workflow。

  ## Why VideoClaw?

  You've tried Sora, Runway, Kling, CogVideo... Each is impressive alone.
  But making a *real* video still means:

  ❌ Writing prompts for each shot manually
  ❌ Waiting, downloading, re-uploading between tools
  ❌ No idea what it costs until the bill arrives
  ❌ Starting from scratch when one shot fails
  ❌ Manually stitching, adding subtitles, music, voiceover

  **VideoClaw fixes all of this.**

  AI agents that think, act, and self-correct. The Director plans,

  locally on your Mac. Auto-fallback to cloud when needed.

  🔀 Smart Model Router

  One pipeline, multiple models. VideoClaw picks the best model
  for each shot based on your strategy: quality, speed, or cost.
